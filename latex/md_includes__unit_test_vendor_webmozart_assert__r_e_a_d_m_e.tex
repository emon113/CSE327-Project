This library contains efficient assertions to test the input and output of your methods. With these assertions, you can greatly reduce the amount of coding needed to write a safe implementation.

All assertions in the {\texttt{ {\ttfamily Assert}}} class throw an {\ttfamily Webmozart\textbackslash{}Assert\textbackslash{}Invalid\+Argument\+Exception} if they fail.\doxysection{FAQ}\label{md_includes__unit_test_vendor_webmozart_assert__r_e_a_d_m_e_autotoc_md626}
{\bfseries{What\textquotesingle{}s the difference to {\texttt{ beberlei/assert}}?}}

This library is heavily inspired by Benjamin Eberlei\textquotesingle{}s wonderful {\texttt{ assert package}}, but fixes a usability issue with error messages that can\textquotesingle{}t be fixed there without breaking backwards compatibility.

This package features usable error messages by default. However, you can also easily write custom error messages\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{Assert::string(\$path, 'The path is expected to be a string. Got: \%s');}

\end{DoxyCode}


In {\texttt{ beberlei/assert}}, the ordering of the {\ttfamily s} placeholders is different for every assertion. This package, on the contrary, provides consistent placeholder ordering for all assertions\+:


\begin{DoxyItemize}
\item {\ttfamily s}\+: The tested value as string, e.\+g. {\ttfamily \char`\"{}/foo/bar\char`\"{}}.
\item {\ttfamily \%2\$s}, {\ttfamily \%3\$s}, ...\+: Additional assertion-\/specific values, e.\+g. the minimum/maximum length, allowed values, etc.
\end{DoxyItemize}

Check the source code of the assertions to find out details about the additional available placeholders.\doxysection{Installation}\label{md_includes__unit_test_vendor_webmozart_assert__r_e_a_d_m_e_autotoc_md627}
Use {\texttt{ Composer}} to install the package\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$ composer require webmozart/assert}

\end{DoxyCode}

\doxysubsection{Extending Assert}\label{md_includes__unit_test_vendor_webmozart_assert__r_e_a_d_m_e_autotoc_md639}
The {\ttfamily Assert} class comes with a few methods, which can be overridden to change the class behaviour. You can also extend it to add your own assertions.\doxysubsubsection{Overriding methods}\label{md_includes__unit_test_vendor_webmozart_assert__r_e_a_d_m_e_autotoc_md640}
Overriding the following methods in your assertion class allows you to change the behaviour of the assertions\+:


\begin{DoxyItemize}
\item {\ttfamily public static function \+\_\+\+\_\+call\+Static(\$name, \$arguments)}
\begin{DoxyItemize}
\item This method is used to \textquotesingle{}create\textquotesingle{} the {\ttfamily null\+Or} and {\ttfamily all} versions of the assertions.
\end{DoxyItemize}
\item {\ttfamily protected static function value\+To\+String(\$value)}
\begin{DoxyItemize}
\item This method is used for error messages, to convert the value to a string value for displaying. You could use this for representing a value object with a {\ttfamily \+\_\+\+\_\+to\+String} method for example.
\end{DoxyItemize}
\item {\ttfamily protected static function type\+To\+String(\$value)}
\begin{DoxyItemize}
\item This method is used for error messages, to convert the a value to a string representing its type.
\end{DoxyItemize}
\item {\ttfamily protected static function strlen(\$value)}
\begin{DoxyItemize}
\item This method is used to calculate string length for relevant methods, using the {\ttfamily mb\+\_\+strlen} if available and useful.
\end{DoxyItemize}
\item {\ttfamily protected static function report\+Invalid\+Argument(\$message)}
\begin{DoxyItemize}
\item This method is called when an assertion fails, with the specified error message. Here you can throw your own exception, or log something.
\end{DoxyItemize}
