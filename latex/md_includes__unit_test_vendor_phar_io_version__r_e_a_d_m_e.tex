 \doxysection{Version constraints}\label{md_includes__unit_test_vendor_phar_io_version__r_e_a_d_m_e_autotoc_md45}
A Version constraint describes a range of versions or a discrete version number. The format of version numbers follows the schema of {\texttt{ semantic versioning}}\+: {\ttfamily $<$major$>$.$<$minor$>$.$<$patch$>$}. A constraint might contain an operator that describes the range.

Beside the typical mathematical operators like {\ttfamily $<$=}, {\ttfamily $>$=}, there are two special operators\+:

{\itshape Caret operator}\+: {\ttfamily $^\wedge$1.0} can be written as {\ttfamily $>$=1.\+0.\+0 $<$2.\+0.\+0} and read as »every Version within major version {\ttfamily 1}«.

{\itshape Tilde operator}\+: {\ttfamily $\sim$1.0.\+0} can be written as {\ttfamily $>$=1.\+0.\+0 $<$1.\+1.\+0} and read as »every version within minor version {\ttfamily 1.\+1}. The behavior of tilde operator depends on whether a patch level version is provided or not. If no patch level is provided, tilde operator behaves like the caret operator\+: {\ttfamily $\sim$1.0} is identical to {\ttfamily $^\wedge$1.0}.\doxysection{Usage examples}\label{md_includes__unit_test_vendor_phar_io_version__r_e_a_d_m_e_autotoc_md46}
